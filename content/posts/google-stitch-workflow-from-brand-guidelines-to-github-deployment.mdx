# Google Stitch Workflow: From Brand Guidelines to GitHub Deployment

I wanted to share some insights about my recent experience with Google Stitch and walk you through the complete workflow I've been using. There are a couple of key aspects I think are worth diving deeper into, especially around how AI can intelligently pull from existing brand guidelines and some workflow considerations when developing on mobile.

## The Google Stitch Design Process

When I first logged into Google Stitch, I kept it simple and asked it to design a site based on a typical CEO thought leadership blog—think Medium-style layouts and functionality. The initial result was... well, let's just say it featured some rather aggressive neon styling that wasn't quite what I had in mind.

But here's where things got interesting.

### Leveraging Existing Brand Guidelines

Instead of manually tweaking colors and fonts, I decided to test Google Stitch's ability to work with existing brand assets. I simply pointed it to [Koto's brand guidelines](https://koto.com/brand), and the AI was able to intelligently extract the appropriate:

- Color palettes
- Typography choices  
- Overall visual styling
- Brand aesthetics

This allowed me to create what essentially became an offshoot of my company's brand identity without having to manually specify every design detail. The AI understood the brand context and applied it consistently across the entire site design.

## The Mobile Development Workflow

This brings me to the second aspect I wanted to discuss: the complete workflow from design to deployment, especially when working entirely from mobile.

### From Stitch to GitHub

Once I had finalized everything in Google Stitch, the platform generated a complete zip file containing all the necessary assets and code. This is why you might notice two different designs embedded in exported files—Google Stitch often provides variations or iterations within the same export.

Here's where the workflow gets a bit interesting when you're working mobile-first:

1. **Design in Google Stitch** - Create and refine the site design
2. **Export to mobile device** - Download the zip file to iPhone Files app
3. **Upload to GitHub** - Use GitHub's web interface to upload the zip file
4. **Agent processing** - My deployment agents automatically detect the new zip file, extract it, and handle the setup

### Workflow Considerations

Working entirely from my hotel room using just my phone highlighted some interesting friction points in the current toolchain. The process of downloading to iOS Files and then uploading via GitHub's web interface works, but it's not exactly seamless.

These are the kinds of workflow quirks I'm constantly thinking about how to improve. The ideal scenario would be direct integration between design tools like Google Stitch and deployment platforms, eliminating these intermediate file management steps.

## Looking Forward

The combination of AI-powered design tools that can intelligently interpret brand guidelines with automated deployment workflows is pretty powerful, even with the current rough edges. Being able to go from concept to deployed site entirely from a mobile device—while maintaining brand consistency—feels like a glimpse into the future of development workflows.

The key is finding ways to smooth out those intermediate steps and create more direct integrations between the tools in our development stack.
