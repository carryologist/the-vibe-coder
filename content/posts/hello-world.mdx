# Hey, welcome in

So this is The Vibe Coder. I've been meaning to start writing about how I actually build software — not the sanitized conference-talk version, but the real stuff. The late-night debugging sessions, the moments where an AI pair programmer suggests something brilliant you never would have thought of, and the quiet satisfaction of deleting code that never needed to exist.

This blog is going to cover a few things I care about deeply: **software development as a craft**, working with AI tools as genuine collaborators rather than fancy autocomplete, building with modern stacks (Next.js, TypeScript, Tailwind, the usual suspects), and what I've started calling "vibe coding" — that flow state where you and your tools are in sync and the code just _feels_ right.

## What even is vibe coding?

It's less of a methodology and more of a mindset. It's when you stop fighting your tools and start leaning into them. It's when you trust the process enough to experiment freely. Here's what it looks like in practice:

```typescript
// Vibe coding means writing the interface you wish existed first,
// then making reality catch up.
interface DevExperience {
  flow: "uninterrupted";
  tools: "working-with-you";
  friction: "minimal";
  curiosity: "high";
}

function startCoding(coffee: boolean): DevExperience {
  if (!coffee) {
    throw new Error("Prerequisites not met.");
  }
  return {
    flow: "uninterrupted",
    tools: "working-with-you",
    friction: "minimal",
    curiosity: "high",
  };
}
```

I don't have a strict posting schedule in mind. I'd rather write fewer things that are actually worth reading than churn out content for the sake of consistency. Expect posts about dev workflows, tool reviews, architecture decisions, things I got wrong, and occasionally some strong opinions held loosely.

If any of that sounds interesting, stick around. And if you want to tell me I'm wrong about something — even better. That's how we all get sharper.
